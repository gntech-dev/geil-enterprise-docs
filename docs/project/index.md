This section controls the GEIL documentation program itself: scope, charter, document inventory, delivery roadmap, and documentation backlog.

## Required use

Before authoring or changing production implementation documentation:

1. Review the project charter for scope and quality rules.
2. Select work from the documentation backlog by priority.
3. Update the document index when creating or materially changing a document.
4. Update the roadmap when a phase deliverable changes state.
5. Update the changelog in the same change set.
